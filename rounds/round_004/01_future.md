In five years, agents handling long-horizon investigations (multi-week scientific lit reviews, multi-month code refactors, multi-year legal-discovery) will need a memory addressing scheme richer than vector-embedding nearest-neighbor retrieval. The interesting bottleneck is not "can I find a related fact" but "can I retrieve facts in the right order, in the right local interpretation, conditional on which workflow path I am currently on." Two different paths through the same conceptual node may need to retrieve different downstream content. Cross-path sharing of a node should also be preserved — when the agent re-enters a familiar conceptual location from a new path, prior context should be available but not collapsed. Embeddings collapse such structure into a single point; graph stores expose nodes but not the path-conditioned interpretation. The next generation of agent memory will need explicitly path-indexed retrieval rather than node-indexed retrieval, with shared-but-locally-reinterpreted nodes as a first-class concept.
