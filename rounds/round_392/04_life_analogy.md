# Life Analogy — Newar bahal courtyard community

The **Newari bahal** is a courtyard-style Buddhist monastery / kinship compound in Kathmandu Valley:
- Square courtyard with surrounding residences + shrines.
- **Guthi** social organisation governs decisions for kinship+religious affairs.
- Decisions made in the **shared courtyard space** where all members can see + speak.
- **Process-level shared context**: members observe each other's daily life + ritual + work.
- Hierarchy is maintained but consensus is prized.

The unique principle: **shared-courtyard process-level context awareness** — agents (community members) share not just the current decision but ALL ongoing operational context, enabling decision via accumulated process visibility.

## Analogical mapping → LLM multi-agent decision

- Bahal courtyard ↔ shared multi-agent operational state store
- Guthi council ↔ multi-agent decision protocol
- Process-level visibility ↔ shared process-context layer for all agents
- Daily observed life ↔ continuous shared-state telemetry

The mechanism: a **shared-courtyard process-context layer** for multi-agent LLMs where ALL agents maintain a shared, append-only process log of their own operational state (tool calls, intermediate beliefs, recent failures). Before making any decision, each agent first reads the SHARED process log; consensus is built atop process-level transparency (not just opinion debate). Differs from voting/debate (opinion-level) by SHARED PROCESS LOG below the opinion layer. Adjacent to Semantic Consensus Framework but distinct in being a CONTINUOUS process log (not a snapshot+conflict-detector).
