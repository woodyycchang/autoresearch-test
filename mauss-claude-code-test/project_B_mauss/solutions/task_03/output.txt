GIVE
----
For any downstream reader or future maintainer:
- The Implementer's exact-string-equality reading of the "target
  itself" exclusion is the only behavioral ambiguity worth flagging
  on review. If a future task spec tightens that wording (e.g.,
  "case-insensitive equality"), the single touch-point is the
  comparison inside `find_anagrams` — not the `_signature` helper.
- The `_signature` helper handles unicode whitespace via
  `str.isspace()`, which is broader than ASCII-only `\s`. Any future
  performance tuning should preserve that semantics.
- No external dependencies were added; stdlib + pytest only.
