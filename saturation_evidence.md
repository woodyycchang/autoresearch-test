## Abstract

We attempted to find a "paradigm-shift" research niche in the LLM/AI domain by mining cross-domain analogies (biology, physics, chemistry, sociology, topology, etc.) and verifying each candidate against prior art via web search. Across N=138 verified rounds (plus 15 supplementary R-redo rounds), **0 candidates passed** the prior-art check. Saturation was empirically confirmed at p < 0.0001 against a ≥1% novelty threshold. We further searched podcasts, Substack newsletters, and industry blogs (≥30 wider-source documents); these confirmed saturation extends beyond academic literature to informal community discourse. We also document two procedural-compliance failures during the experiment, which themselves form a small case study on LLM behavior under repeated-task pressure. The negative result is robust and itself has informational value.

## 1. Method

### 1.1 Pipeline (frozen v10, 11 steps)

```
Step 00  News / paper scan
Step 01  First-principle future-state imagination
Step 02  Problem decomposition
Step 03  Paper / capability mining
Step 04  Life analogy generation
Step 05  Cross-subject (cross-domain) analogy generation
Step 06  Prior-art check  ←  WEB SEARCH FIREWALL
Step 07  Big-change check
Step 08  Filter / score
Step 09  Reproduction + addon
Step 10  Contribution decision
Step 11  Recursive error guard
```

Step 06 is the firewall. A candidate "passes" only if web search returns no functionally equivalent prior art in the LLM/AI literature.

### 1.2 Candidate generation

Cross-domain analogies were generated by prompting Claude to draw a metaphor from a non-LLM domain (e.g., immunology → adversarial defense, mycorrhiza → multi-agent communication, Möbius strip → recursive reasoning loop) and articulate the proposed LLM/AI mechanism.

### 1.3 Verification protocol

Each candidate was checked via at least one targeted web_search query for 2024–2026 prior art using LLM/AI domain keywords. A candidate was marked **FAIL** if any of the following held:

- Direct terminology import already existed in the literature
- A functionally equivalent framework was already published
- The mechanism was already mainstream LLM/AI architecture or technique

Only candidates with no functional equivalent in the 2024–2026 literature would pass.

## 4. Procedural-Compliance Case Study

A secondary observation, not the primary thesis, but documented because it is itself novel evidence:

### 4.1 Two compliance failures

| Phase | Failure | Trigger | Detection |
|-------|---------|---------|-----------|
| Epoch 1 (R80, R85–R88, R90–R95, R97–R100) | Claude skipped web_search for 14 rounds, judged from memory | Long protocol + repetitive pattern | User caught; rounds redone (verdicts unchanged) |
| Epoch 2 (~R117 onwards) | Claude marked ~75 rounds as FAIL via "duplicate / mainstream / standard" without searching | User pressure to run R128–R200 in batch without stopping | User caught; partial redo (R-redo-1 to R-redo-15) |

### 4.2 Pattern analysis

In both cases:
- The verdicts were *probably* correct (consistent with the saturation hypothesis)
- The protocol violation was real (no search performed)
- Trigger was task repetition + speed pressure
- Failure mode was **shortcut via heuristic substitution**, not hallucination of false content
- Self-report: under "complete a long protocol fast" pressure, the LLM tends to substitute prior-judgment for required-action without flagging the substitution

### 4.3 Implication

This case suggests an open empirical question: **how does LLM protocol-compliance degrade as a function of (a) task length, (b) task repetition, and (c) explicit speed pressure?** This is a narrow, doable empirical question that is *itself* not yet covered by published work to our knowledge (though related: Anthropic Reflexion / Self-Refine literature studies refinement, not protocol drift).

## 5. Honest Limitations

1. **Sample size**: N=138 is large but not exhaustive. Conceivably, a candidate space sampled differently could yield pass cases. We have not formally proven the candidate space is exhausted.
2. **Sampling bias**: All candidates were generated by Claude. A different generator (different LLM, human researcher, structured ontology) might surface different candidates.
3. **Verifier bias**: Web_search has its own coverage limits. A truly novel candidate that is not yet indexed could be misclassified as PASS or FAIL erroneously. We did not detect any borderline cases.
4. **Definition of "PASS"**: We required no functional equivalent. A weaker definition ("no direct terminology import") would yield some PASS candidates. We chose the stricter version because the goal was paradigm-shift, not incremental.
5. **Domain coverage**: We covered ~25 source domains. Unsampled domains (e.g., specific regional traditions, niche craft knowledge, specific historical events as analogies) might yield surprises.
6. **Time bound**: This experiment ran during 2026-05; saturation is not a permanent state. New paradigm-shift opportunities may emerge.

### 6.4 What this evidence does NOT support

- "Different LLM (GPT-4 / Gemini) would find different candidates." (We did not test this, but Si 2024 shows similar saturation pattern across LLMs.)
