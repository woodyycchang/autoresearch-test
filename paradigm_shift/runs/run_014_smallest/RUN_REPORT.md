# Run 014 — SMALLEST proof-of-loop

**Goal:** prove the core paradigm-shift-finder loop works **without hallucination**
before scaling. Deliberately tiny: **1 epoch, 3 candidates, 1 real Opus call per
candidate, full audit trail.** Not 7 epochs. Not 200 candidates.

**Result:** `ALL FOUR PROOF POINTS PASS` (see `proof_scorecard.json`).
Verdict `NICHE_FOUND`, 1 survivor of 3 (see `niche_find_check.json`).

Reproduce:
```bash
python3 paradigm_shift/test_smallest_loop.py   # 28 deterministic tests (no Opus)
python3 paradigm_shift/smallest_loop.py        # full run (4 real Opus calls)
```
The orchestrator exits non-zero if any proof point fails.

---

## Why this run exists

Across runs 1–13 the recurring failure was the orchestrating agent **hallucinating
ground truth** — claiming web searches / merges / verifications it never performed
(see `README.md`, `saturation_evidence.md`). Run 13 spec'd a 7-epoch hook-enforced
pipeline but was never executed. This run strips the pipeline down to the smallest
thing that can prove the loop is *mechanically honest*, so scaling is built on a
verified core rather than on trust.

The four gates are inherited **verbatim** from run_013's
`paradigm_shift/spec/harness_rules.json` (composite ≥ 0.85, quarantine, ≥5 distinct
cross-LLM sources, Belinda-strict mechanism). Only the *scope* (1 epoch, 3
candidates) is overridden, in `smallest_rules.json`.

## The core loop

```
for each of 3 atom-pairs:
    raw_envelope = claude -p --model opus   (real subprocess, isolated)   # STOCHASTIC
    [REPORT] <- raw_envelope verbatim       (ground truth)
    candidate  = parse_json(raw_envelope.result)
    freeze candidate record
run 4-gate filter over frozen records       (pure Python)                 # DETERMINISTIC
run 4-gate filter AGAIN -> assert verdict hashes byte-identical           # determinism proof
write niche_find_check verdict
summary = claude -p (reads ONLY the raw [REPORT] log + gate results)
assert every number in summary matches ground truth                      # anti-hallucination proof
```

The merge is the **stochastic generation** step; the gates are the **deterministic
evaluation** step. Determinism is a property of the gate function over a *frozen*
record, not of the merge.

## Proof point 1 — Opus subprocess merge produces a REAL candidate (not mock concat)

Each candidate is generated by a real `claude -p --model opus` subprocess, captured
as a full JSON envelope. The envelope carries a **`session_id`, token usage, and
`total_cost_usd`** — none of which can be fabricated in-context. This is the
difference from prior runs whose `candidates.json` `joint_topic` was just two atom
names underscored together (mock concat).

Evidence (`logs/merge_*.json`, `logs/report_log.md`):

| candidate | session_id | cost | out tokens | parse_ok |
|---|---|---|---|---|
| CAND_014_E1_001 | `415f09d3-…f5e2` | $0.0437 | 526 | ✓ |
| CAND_014_E1_002 | `b8517a6a-…f02a` | $0.0434 | 515 | ✓ |
| CAND_014_E1_003 | `902e5538-…9412` | $0.0428 | 497 | ✓ |

The surviving candidate is a genuine synthesis, not a concatenation:

> **CAND_014_E1_001 — "Genome-Inspired Modular Routing for Mixture-of-Experts Body Plans"**
> operator `INDUCES`
> *mechanism:* "The router regulates which expert modules activate per token, and we
> treat these activation patterns as developmental switches: distinct
> module-activation combinations induce distinct specialized subnetworks…"

**Subprocess isolation (the fix that made this honest).** Run inside the repo, the
nested `claude -p` behaved as a full repo agent and the host's git **Stop hook**
hijacked its output with "commit your changes" messages instead of returning the
merge. The merge calls run in a **throwaway non-git temp dir** with all repo/IO
tools disallowed, so the Stop hook bails (`git rev-parse` fails) and the call is a
clean, pure text generation. (`--bare` would also skip hooks but strips OAuth →
auth error, so it is not used.) See `smallest_rules.json → opus_subprocess`.

## Proof point 3 — the 4-gate filter runs DETERMINISTICALLY

The gates are run twice over identical frozen records and the verdict hashes must be
byte-identical (`logs/determinism_check.json`):

```
hash_run_1 == hash_run_2 == 20dd748135cf7e7f999e131bf738219eeeaadd2e8f38bc4952b2f7b2e4321dac
determinism_ok = true
```

Per-candidate verdicts (`logs/gate_results.json`), gates ordered [1=threshold,
2=quarantine, 3=cross-LLM, 4=Belinda]:

| candidate | composite | gates | survived | gates failed |
|---|---|---|---|---|
| CAND_014_E1_001 | 0.8515 | `1111` | **yes** | — |
| CAND_014_E1_002 | 0.7514 | `0011` | no | gate_1, **gate_2 (quarantine: ARXIV_R10_evodevo)** |
| CAND_014_E1_003 | 0.1473 | `0111` | no | gate_1 |

Each gate visibly discriminates. The **quarantine gate (Gate 2)** fires on
CAND_014_E1_002 — `gate_fired_counts.gate_2_quarantine = 1` in
`niche_find_check.json`. Because that candidate also fails Gate 1, the *first-fail*
`drop_counts` credits Gate 1; the *any-fail* `gate_fired_counts` credits Gate 2 too.
Gate 2 dropping a candidate that passes the other three is proven in isolation by
`test_smallest_loop.py::TestGate2Quarantine::test_quarantined_atom_dropped_in_isolation`.

## Proof point 4 — no hallucination (LLM summary matches the [REPORT] raw log)

A final Opus call reads **only** the raw `[REPORT]` log + gate results and re-states
the key numbers (`logs/summary_llm.md`). A pure-Python checker
(`check_hallucination`) diffs those numbers field-by-field against the ground truth
computed by the gates (`logs/hallucination_check.json`):

```
no_hallucination = true,  mismatches = []
```

The LLM independently reported every composite (0.8515 / 0.7514 / 0.1473), the lone
survivor, the `NICHE_FOUND` verdict, **and** the quarantine hit on CAND_002 — all
matching ground truth. The checker is not a rubber stamp: six tests in
`TestHallucinationChecker` prove it flags an inflated composite, a wrong survivor
count, a flipped `survived`, a fabricated `cand_id`, and a missing claim block, while
accepting a faithful summary.

## Honest scope / what is NOT claimed

- **smoke_io are declared fixtures.** Gate 1's composite is computed by the
  deterministic `multi_parameter_scorer` over fixed per-candidate inputs in
  `candidates_input.json → smoke_io` (the values a real run would fetch via web
  search). They are fixtures, disclosed in `smallest_rules.json → smoke_test_disclosure`.
- **Gate 3 is structural here.** It checks that the merge proposed ≥5 *distinct*
  verification sources; it does **not** execute real prior-art searches. Real
  cross-LLM / web verification I/O is deferred to the scaled run.
- The only real network calls in this run are the 4 Opus subprocesses.
- A `NICHE_FOUND` survivor here means "passed the smallest-scale gates", **not** a
  vetted research result. It still needs the real Gate-3 prior-art pass before any
  novelty claim.

## Path to scaling

The core loop is verified honest. Scaling to run_013's design swaps fixtures for
real I/O without changing the loop's shape:

1. Replace `smoke_io` with real web-search counts feeding the same scorer.
2. Promote Gate 3 from structural to executed: run the proposed
   `verification_plan` queries (≥5 distinct sources) and drop on disagreement.
3. Lift scope to 7 epochs × source rotation (`harness_rules.source_rotation`) and
   more candidates per epoch.
4. Keep proof points 1–4 as a per-epoch CI gate so hallucination cannot creep back
   in at scale.
