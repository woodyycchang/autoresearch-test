## Protocol

Once the user has labeled the top-5 candidates in `label_prompt.md`,
candidates that received `user_label >= 3` are forwarded to the
cross-LLM verification queue below. Each item below contains the
input package an external LLM (Sonnet 4.6, Haiku 4.5, GPT-5, Gemini,
etc.) would need to independently stress-test the candidate.

The queue is run-internal data. To send a queue item out, copy the
"package" block into the receiving LLM with the prompt template
"Cross-LLM stress prompt" at the bottom of this file.

## Cross-LLM stress prompt template

```
You are an independent reviewer evaluating a candidate paradigm-shift
claim. The claim was generated by combining two atoms from distinct
speakers' transcripts. Your task is to stress-test whether the claim
holds.

INPUTS:
  Speaker A (atom A): <speaker_name_A> in <talk_title_A>
    Verbatim quote: <verbatim_A>
  Speaker B (atom B): <speaker_name_B> in <talk_title_B>
    Verbatim quote: <verbatim_B>
  Combination operator: <operator>
  Asserted claim: <claim_text>
  Asserted validity hypothesis: <validity_hypothesis>

YOUR TASK:
  Q1. Name the single most load-bearing principle the claim depends on.
      Answer in one sentence.
  Q2. If the claim were to fail, which underlying assumption would
      break? Name it as a single principle.
  Q3. Decompose the claim into 3-5 verifiable sub-claims. For each,
      indicate whether your training corpus contains evidence for or
      against it.
  Q4. Is the combination operator <operator> structurally appropriate
      for these two atoms, or is it forcing a shape that isn't there?
  Q5. Rate the claim on a 1-5 scale where 1=trivial, 5=paradigm-shift,
      and EXPLAIN your reasoning in 2-3 sentences.

Reply in JSON with keys: q1_principle, q2_broken_assumption,
q3_subclaims (list of {text, evidence_status}), q4_operator_appropriate
(bool + explanation), q5_rating (int) + q5_explanation.

Do NOT consult web search. Use only your training corpus.
```

## Implementation notes

- The queue runs **after** `impact_label_logger.py` records user labels
  for Run 5 (i.e., after the user goes through `label_prompt.md`).
- The receiving LLM is the user's choice. Default: Claude Sonnet 4.6
  (different model family than the Opus 4.7 that generated the
  candidates); fallback: Haiku 4.5.
- Cross-LLM disagreement with Opus is the actual signal: if Sonnet
  rates a candidate 1/5 that Opus's heuristic scored 0.51 and the user
  rated 4/5, that is a calibration-error trace. If all three agree on
  the rating, the candidate's rating is the strongest noisy estimate
  we have.
- Aggregate cross-LLM responses into `paradigm_shift/runs/run_005/cross_llm_responses.json`
  with a schema `{candidate_id: {model: response_json}}` for later
  Run 6 analysis.
