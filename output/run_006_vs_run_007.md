# Paradigm-Shift Finder — Run 6 vs Run 7

> Comparison memo. Run 7 introduces two precision-first filters —
> `semantic_coherence_check` and `atom_quality_filter` — and re-runs on
> Run 6's purified 13-transcript corpus.

## What Run 7 is reacting to

Run 6 ended with 14 surviving candidates. Reading them back showed a
consistent failure mode: every survivor was a **surface combination**
of a speaker's illustrative analogy with another speaker's unrelated
open problem. The deterministic combinator template
("Apply analogy X to problem Y") fires on any (analogy_atom,
open_problem_atom) pair regardless of whether X's source domain has
any mechanical relationship to Y's target domain. Examples:

| CAND | Pair (paradigm types) | Why it shouldn't have shipped |
|---|---|---|
| `_011` | `area under curve` × `the leap to unsolved math` | startup-product metaphor pinned to a math-research open problem |
| `_014` | `handwritten digits` × `the leap to unsolved math` | illustrative ML aside vs. math-research framing |
| `_017` | `like a phone book` × `world / user / self models` | memorisation metaphor vs. self-model agenda |
| `_018` | `conservation of happiness` × `the leap to unsolved math` | startup-philosophy quip vs. math-research framing |
| `_019` | `pink qualia / elephantine` × `world / user / self models` | consciousness aside vs. ML agenda |

The arXiv gate (Run 6's main contribution) caught some low-quality
sub-claims but had nothing to say about the underlying pairing — every
sub-claim was already "about an LLM", which made the literature search
trivially return *some* arXiv paper.

## Two new filters in Run 7

### 1. `paradigm_shift/atom_quality_filter.py`

Rejects atoms whose verbatim quote is dominated by talk language:

- **`REJECT_FRAGMENT`** — no finite verb after stripping discourse
  markers, or trailing ellipsis ("...").
- **`REJECT_TALK_STAMMER`** — repeated tokens or a repeated 3-gram
  inside the quote (auto-caption stammer like "I I I" or "he can
  read a he can read like a phone book").
- **`REJECT_TALK_FILLER`** — filler density ≥ 0.20 ("I think the way
  I think of these is I think they're two very basic prim…").
- **`REJECT_ILLUSTRATIVE_ANALOGY`** — analogy whose vehicle hits a
  closed list of mundane objects (area under the curve, phone book,
  conservation of happiness, like-a-fab, …).
- **`REJECT_NO_TECHNICAL_CONTENT`** — no domain-vocabulary noun
  (ml_research / consciousness_brain / hardware_compute /
  startup_business / math_formal) and no quantitative claim.

### 2. `paradigm_shift/semantic_coherence_check.py`

For every (atom_a, atom_b) candidate:

- **Cosine similarity** of TF-IDF vectors over stemmed content words.
  Threshold `0.30` as per spec. (sentence-transformers was the
  preferred encoder, but HuggingFace is blocked by the environment's
  network policy. TF-IDF on 1-2-sentence quotes saturates well below
  0.30 even for clearly same-topic pairs, so a **Jaccard fallback**
  at `0.20` is also computed.)
- **Domain classifier** by keyword vote across 5 named domains. A
  candidate is `REJECT_DOMAIN_MISMATCH` when both atoms vote ≥ 2 for
  *different* domains and share no bridge term (scaling, bottleneck,
  constraint, …).
- **`ACCEPT`** requires: (cosine ≥ 0.30 OR jaccard ≥ 0.20 OR both
  atoms confidently in the same named domain) AND no confident
  cross-domain mismatch.

## What this comparison tells us

1. **Talk-language is the dominant atom-level failure mode in
   auto-captioned tech-leader transcripts.** Of 734 paradigm atoms,
   53.4% get rejected by `atom_quality_filter`, and the majority of
   those rejections are not for being conversational *per se* but for
   containing zero domain-vocabulary nouns. The regex tagger has high
   recall and very low precision on this material — it needs a quality
   gate downstream, not a smarter regex.
2. **Surface combinations are a candidate-level failure mode the
   arXiv gate cannot catch.** All 14 of Run 6's survivors had an arXiv
   citation for each sub-claim, yet every one of them was a pairing
   of two unrelated atoms. The `semantic_coherence_check` correctly
   rejects 14 / 14 of them when applied retrospectively.
3. **Final survivor count drops 14 → 7, and the surviving pairs are
   all genuine ML pairs.** Every one of the 7 survivors places an
   atom from one ML transcript next to an atom from another ML
   transcript on a topic where literature already exists (RLHF,
   self-improvement, long-context alignment, LLM inference memory
   bandwidth, RL data scarcity). Whether the *combination* is novel
   is a downstream question — Run 7 only claims the pairing is now
   mechanically coherent.

## Open issues (Run 8 candidates)

- **TF-IDF is the wrong tool**, even with stemming. Two ML atoms with
  paraphrased vocabulary score cosine ≈ 0 unless they share exact
  tokens. The Jaccard fallback partially compensates, but on average
  the *same-domain-confident* branch is doing most of the actual
  accepting. Unblocking HuggingFace and swapping in a real sentence
  encoder would let us drop the Jaccard branch and the domain
  classifier could become a sanity check rather than the load-bearing
  signal.
- **`PREDICTION_GROUNDED_IN_PRINCIPLE` and `BLOCKER_DISSOLVED_BY_PRINCIPLE`
  both pair against T007/S009 in 3 of the 7 survivors.** That one atom
  ("assistant model … through the following process") is a hub. Either
  it is the most-loaded "first_principle" atom in the corpus, or the
  combinator is over-weighting it. Run 8 should diversity-cap pairings
  per atom.
- **The arXiv-overlap-≥2 rejection of `_039` and `_042` is a
  *precision* signal we should be capturing.** The matched papers
  *are* arXiv-resident and on-topic for the query, but on-closer-look
  they don't actually overlap the atom vocabulary. A learned check
  ("does this abstract resolve this atom?") would do better than the
  current keyword overlap heuristic.
